## 📋 Hallazgos pendientes (priorizados)

### 🔴 HIGH — recomiendo arreglar pronto

1. **Polling timers sin cleanup en visibilitychange (DESCARTADO)**
   `popup.js:1296, 4550, 5315`
   `setInterval(refreshUsage, 60_000)` y similares se crean en DOMContentLoaded pero nunca se clearInterval. Abrir/cerrar popup repetidamente acumula timers.
   **Fix:** Guardar timerId, clearInterval en visibilitychange:hidden, re-crear en visible.

2. **Prospect lock race condition (check + lock no atómico)**
   `popup.js:5940-5945`
   Hay gap entre `getActiveProspectLock()` y `lockProspect()`. Otro MB puede entrar.
   **Fix:** Mover la verificación adentro de lockProspect (transaction) o usar UPSERT con WHERE para garantizar atomicidad.

3. **Faltan índices en Supabase para queries frecuentes**
   `toolbar_api_usage`, `toolbar_historial`, `toolbar_pitch_feedback`, `toolbar_sendtrack`.
   Queries como `?user_email=eq.X&day=gte.Y` hacen full table scan.
   **Fix SQL** (correr cuando vuelvas):
   ```sql
   CREATE INDEX IF NOT EXISTS idx_api_usage_user_day  ON toolbar_api_usage(user_email, day);
   CREATE INDEX IF NOT EXISTS idx_historial_buyer     ON toolbar_historial(media_buyer);
   CREATE INDEX IF NOT EXISTS idx_pitch_feedback_user ON toolbar_pitch_feedback(user_email);
   CREATE INDEX IF NOT EXISTS idx_sendtrack_buyer    ON toolbar_sendtrack(media_buyer);
   CREATE INDEX IF NOT EXISTS idx_review_queue_created_by_status ON toolbar_review_queue(created_by, status);
   CREATE INDEX IF NOT EXISTS idx_review_queue_status_date ON toolbar_review_queue(status, created_at DESC);
   ```

4. **Admin panel queries leen de `toolbar_historial` con columnas mal nombradas**
   `popup.js:677-737` — query usa `user_email=eq.X&fecha=gte.Y` pero la tabla tiene `media_buyer` y `date`. Worker autopilot ni siquiera escribe ahí.
   **Fix grande (~1h):** refactor `loadAdminActivity` para leer de `toolbar_review_queue` agrupado por `created_by`. Esa tabla es la fuente real (autopilot + CSV + Monday + manual). Ya lo recomendé — lo dejo para sesión sincrónica porque cambia varias funciones de render (Leaderboard, MBSummaries, ByUser).

5. **Bulk validation Prospects (acción más pedida)**
   No existe. Cada lead requiere click individual ✅/❌. Para 50 leads/día son ~250 clicks.
   **Fix:** Agregar checkboxes en cards + botón "Validar seleccionados" en header.

### 🟡 MED

6. **Apollo + RapidAPI duplicados entre worker y popup**
   Si autopilot enriquece de noche y el MB analiza el mismo dominio al día siguiente, Apollo se llama 2 veces.
   **Fix:** Cachear Apollo result en `toolbar_traffic_cache` (o tabla nueva `toolbar_apollo_cache`) con TTL 7d.

7. **Embeddings (Voyage) re-computados para mismo input**
   `popup.js:2620, 2645` — cada like/dislike re-embebe sin chequear si ya existe.
   **Fix:** SELECT existing embedding for same context_hash antes de llamar Voyage.

8. **renderProspectCard innerHTML rewrite completo**
   `popup.js:5566` — re-render full DOM cada refresh aunque solo cambió 1 card.
   **Fix:** Diff por id, replace solo cards modificadas. Mediano effort.

9. **getTraffic await sin AbortController**
   `popup.js:5954` — al cerrar/re-abrir card rápido, fetch en flight pisa data.
   **Fix:** AbortController por domain.

10. **Modal admin sin z-index explícito**
    `popup.css:504, 507` — modales tapan admin panel.
    **Fix:** `.modal { z-index: 99 }` `.admin-panel { z-index: 100 }`.

11. **chrome.storage.local guarda accessToken plaintext**
    Riesgo en máquinas compartidas.
    **Fix:** Mover accessToken a `chrome.storage.session` (vive solo durante sesión browser).

12. **Auto-refresh re-análisis sin debounce**
    `popup.js:223` — 3 navegaciones rápidas → 3 análisis paralelos.
    **Fix:** AbortController para cancelar análisis previo.

13. **Apollo reveal button sin bounds check**
    `popup.js:2588` — `people[idx]` puede ser undefined post re-render.
    **Fix:** Guard `if (!person) return;`

14. **Toast queue serializado**
    `popup.js:1075` — toasts paralelos se solapan. Quick win ya aplicó anti-overlap >3, pero idealmente queue serial.
