ADEQ Toolbar — Privacy Policy

Last updated: April 20, 2026 · Contact: mgargiulo@adeqmedia.com

1. Who this applies to

ADEQ Toolbar is a private Chrome extension used exclusively by authorized employees of ADEQ Media for internal B2B prospecting. Access is restricted by email whitelist to:

  • mgargiulo@adeqmedia.com
  • sales@adeqmedia.com
  • dhorovitz@adeqmedia.com

The extension is not intended for, nor accessible to, users outside ADEQ Media.

2. Data the extension handles

From the employee (you)

  • Authentication: your Supabase email/password login, Google OAuth access token used only to send Gmail on your behalf.
  • Local preferences: session cache, recent analyses, keyword lists stored in chrome.storage.local.

From the websites you analyze

  • URL, page title, meta description, visible ad network scripts, banner sizes.
  • Public WHOIS records and contact-page emails of the analyzed domain.
  • Aggregated traffic metrics from SimilarWeb (via RapidAPI).
  • Decision-maker names and business emails from Apollo.io (when available under the account's plan).

3. Where data goes

  • Monday.com — prospect records are pushed to ADEQ Media's private CRM board.
  • Supabase — authentication, shared config, activity logs, audit trail.
  • Gmail (via Google OAuth) — sending prepared sales emails from the authenticated employee's account.
  • Third-party APIs called to fetch public information about websites: SimilarWeb (RapidAPI), Apollo.io, Gemini (Google), public WHOIS (who.is), website.informer.com, DNS-over-HTTPS (dns.google).

4. What we do NOT do

  • We do not sell or transfer user data to third parties.
  • We do not use data for advertising, profiling, credit scoring or loan decisions.
  • We do not collect browsing history outside the websites actively opened by the employee while using the toolbar.
  • We do not run any remote JavaScript. All code is packaged within the extension.

5. Permissions explained

  • activeTab, tabs, scripting — read the URL, title, and meta of the website being analyzed; open selected prospect URLs in background tabs.
  • storage — local cache of session token and recent lookups.
  • identity — Gmail OAuth to send emails from the employee's account.
  • sidePanel — display the toolbar UI as a side panel.
  • <all_urls> host permission — the team analyzes any publisher site, domain cannot be predicted in advance.

6. Data retention

  • Supabase records are kept for the lifetime of the ADEQ Media account.
  • Local caches expire between 24 hours and 60 days depending on type.
  • On uninstall, local data is removed automatically by Chrome.

7. Security

  • All API calls use HTTPS.
  • Supabase tables use Row-Level Security (RLS) restricted to authenticated ADEQ Media accounts.
  • Gmail identity is verified against the logged-in Supabase email before any email is sent.
